Introducción

Descripción General

Este sistema está diseñado para trabajar con HTML, CSS y JavaScript, sin depender de bibliotecas de estilos externas. Las clases y selectores utilizados en el sistema incorporan el prefijo "ec-" de Enerclic para evitar conflictos con las nomenclaturas comunes utilizadas por bibliotecas populares como Bootstrap y otras. De todas maneras, en caso de utilizar este sistema junto con otros frameworks, se debe prestar especial atención a posibles contingencias, como la sobreescritura de clases, para garantizar una integración sin problemas.

  • Guías de Estilo para Componentes:

    Se proporciona un conjunto coherente de estilos y clases que se pueden aplicar a componentes y elementos con el fin de ayudan a mantener un diseño visual uniforme en toda la aplicación.

  • Guías y Sugerencias de Funcionalidades del Frontend:

    Además de los estilos, se ofrecen guías y sugerencias para implementar funcionalidades comunes en el frontend. Estas pautas son especialmente útiles para el desarrollo de aplicaciones desce cero, pero también pueden aplicarse a proyectos existentes que busquen mejorar su diseño y funcionalidad.

Para simplificar la implementación, puedes acceder a un proyecto de ejemplo que sirve como plantilla base. Encontrarás el código completo y la estructura en el siguiente enlace. Este proyecto de ejemplo puede ser un punto de partida sólido, ya que contiene la estructura básica y con funciones reutilizables que pueden servir de guia para trabajar de manera consistente.

JAVASCRIPT
toggleAdvanceContent()
Esta función se encarga de mostrar u ocultar contenido avanzado en tu web cuando se hace clic en un "trigger" correspondiente. Para que funcione correctamente, es esencial que sigas una convención de nomenclatura para los IDs de los elementos HTML. A continuación, se describen las clases y IDs requeridos:
Requerimientos
  • El trigger deben tener class="advance-trigger"
  • El trigger debe tener id="advance_trigger-form1" la terminacion form1 es reemplazable por cualquier termino, pero es recomendable que sea el ID del formulario en el que se encuentra, para mayor claridad.
  • La flecha debe tener el id="advance_arrow-form1"
  • El contenido avanzado debe tener el ID="advance_content-form1"
  • Es necesario incluir en css las clases .hideElement y .spin 
const advanceTriggers = document.querySelectorAll('.advance-trigger');

const toggleAdvanceContent = () =>{
    advanceTriggers.forEach(trigger => {
        trigger.addEventListener("click", () => {
            const form = trigger.id.split("-")[1]
            const container = document.getElementById(`advance_content-${form}`)
            const arrow = document.getElementById(`advance_arrow-${form}`)
            container.classList.toggle("hideElement")
            arrow.classList.toggle("spin")
        })
    })
}

Rutas

Este es un sistema para manejar rutas desde el Frontend, que muestra y oculta secciones al seleccionar tabs, y mantiene la tab seleccionada con el color de Enerclic.

Requerimientos

  • Las tabs deben tener la clase class="ec-sidebar__tab".
  • Cada tab debe tener de href # + el ID de la seccion que se quiere mostrar ej. href="#form1".
  • Las secciones que se muestran y ocultan deben tener la clase class="ec-section".

JAVASCRIPT
const sections = document.querySelectorAll('.ec-section');

const hideAllsections = () => sections.forEach(section => {
    section.classList.add("hideElement")
});
const selectTab = (sectionId) => {
    const tabs = document.querySelectorAll('.ec-sidebar__tab');
    tabs.forEach(tab => {
        if (tab.getAttribute('href') === `#${sectionId}`) {
            tab.classList.add('.ec-sidebar__selected-tab');
        } else {
            tab.classList.remove('.ec-sidebar__selected-tab');
        }
    });
}
const showSectionForCurrentRoute = () => {
    const currentRoute = window.location.hash;
    const sectionId = currentRoute.substring(1);
    hideAllsections();
    selectTab(sectionId);
    const sectionToShow = document.getElementById(sectionId);
    if (sectionToShow) {
        sectionToShow.classList.remove("hideElement");
    }
}

// Si no hay una ruta en la URL establece la sección que se muestra al iniciar la web
if (window.location.hash === "") {
    window.location.hash = "form1";
}

// Llamar a showSectionForCurrentRoute() al cambiar de ruta y al iniciar la web
window.addEventListener('hashchange', showSectionForCurrentRoute);
showSectionForCurrentRoute();

Manipulación de Datos

La visualización y modificación de información por parte del usuario a través de formularios, puede variar en cada aplicación y depende en gran parte del backend y las características específicas de la aplicación, por lo que no se abordan aquí métodos de conexión con la base de datos ni solicitudes a la API. No obstante, se presentan funciones reutilizables diseñadas para preparar los datos de manera adecuada antes de su envío.

Requerimientos
  • Los campos deberán tener como clase el ID del formulario al que pertenecen + "-field" Ej. class="form3-field".
  • Los campos deberan tener como ID la key con la que viene el dato correspondiente en el GET. Ej const obj = { "dato1" : "Dato 1"} id="dato1"
  • Este sistema funciona únicamente en casos donde no se requieren conversión de tipos, ni calculos, ni filtrados, desde el front. En estos casos debera modificarse setFormValues() definiendo el valor de cada campo de a uno. Está sistematizado para que corresponda un string o boolean por cada campo, el cual se recibirá y enviará de la misma manera. En caso de que un campo no necesite recibir un valor, se pintará vacio, y en caso de que el endpoint del GET de un dato que no se utilice en ningun campo se lo ignorará.

En el siguiente ejemplo puedes ver el resultado de la implementación. Contiene un console.log con la informacion que se envía al clickear en "Save".

Traducciones

Este es un sistema de traducción sin bibliotecas externas. Es recomendable incluir todos los textos en un data-lang y no dejarlos declarados simplemente en el HTML. Muchas palabras parecen no necesitar traducción, por ejemplo "IP" o "Router" probablemente se elija dejar así tanto en español como en inglés, pero es ideal que sea incluido en el data-lang igualmente, para ser consistente y poder traducir en otros idiomas en el futuro.

Requerimientos

  • Todos los elementos cuyo contenido sea texto deben tener como atributo data-lang="nombre_a_eleccion". Utilizar un nombre corto descriptivo del contenido. Ej. <p data-lang="form"></p>
  • Todos los inputs de tipo button deben tener como atributo data-lang-value="nombre_a_eleccion" Ej. <input type="button" data-lang-value="save">
  • Todos los elementos que lleven tooltips deben tener como atributo data-lang-tooltips="nombre_a_eleccion" Ej. <p data-lang="info" data-lang-tooltips="info_msg"></p>
  • Definir objetos para cada idioma que tenga como key los data-lang, data-lang-value y data-lang-tooltips, y como value el texto correspondiente. Ej. const en = { "form": "Form", "Info": "information" "Info_msg": "This is the info section", "save": "Save" } const es = { "form": "Formulario", "Info": "información" "Info_msg": "Esta es la sección de información", "save": "Guardar" }

const applySelectedLang = () => {
	const langTags = document.querySelectorAll('[data-lang]');
	const langPlaceHolder = document.querySelectorAll('[data-lang-placeholder]');
	const langToolTips = document.querySelectorAll('[data-lang-tooltips]');
	const langValue = document.querySelectorAll('[data-lang-value]');

	langTags.forEach((tag) =>  tag.textContent = lang[tag.getAttribute('data-lang')]);
	langPlaceHolder.forEach((tag) => tag.placeholder = lang[tag.getAttribute('data-lang-placeholder')]);
	langToolTips.forEach((tag) => tag.title = lang[tag.getAttribute('data-lang-tooltips')]);
	langValue.forEach((tag) => tag.value = lang[tag.getAttribute('data-lang-value')]);
}

const setLang = () => {
	currentLang = document.getElementById("select-lang").value.toLowerCase();
	lang = languages[currentLang];
	applySelectedLang();
}
